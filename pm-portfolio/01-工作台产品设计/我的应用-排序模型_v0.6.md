## 📦 核心变更总览

| 变更项 | 旧（v0.5） | 新（v0.6） |
|--------|-----------|-----------|
| **排序模型** | 三区：置顶 → 频率 → 补齐 | 两区：置顶 → 混排（纯频率） |
| **收藏定位** | 参与排序（频率区 = 收藏+默认模板） | 纯标记 ⭐，不参与排序 |
| **默认初始化** | 管理员批量收藏 + 批量置顶 | 管理员批量置顶（不初始化收藏） |
| **卡片内容** | 收藏+默认模板 | 置顶 + 全部权限可见（按频率排） |
| **管理弹窗** | 三区列表 + 添加按钮 | 两tab + 规则说明，去掉添加按钮 |
| **应用中心** | 全部应用只能收藏 | 全部应用可收藏 + 直接置顶 |

> **不变 → 常：** 置顶区（人定顺序）在最前面，不可变。

## 1. 排序模型

### 1.1 两区模型

```
卡片展示顺序（自上而下）：

┌─────────────────────────────────────────────┐
│ 📌 置顶区（人定的）                         │
│ · 管理员默认置顶 + 用户手动置顶            │
│ · 用户拖拽排序，不限制数量                  │
├─────────────────────────────────────────────┤
│ 📊 混排区（数据定的）                       │
│ · 用户权限可见的全部应用                    │
│ · 统一按访问频率降序自动排列，不可拖拽      │
│ · 收藏的应用带 ⭐ 角标，不影响排序           │
│ · 卡片固定 6 行 + 滚动，不设数量上限        │
└─────────────────────────────────────────────┘
```

### 1.2 收藏的新定位

| 维度 | 说明 |
|------|------|
| **不参与排序** | 混排区纯按频率，收藏不带来任何排序加成 |
| **⭐ 标记** | 卡片上带 ⭐ 角标，用户看一眼就知道"这是我标记过的" |
| **置顶快捷入口** | 管理弹窗中「我的收藏」tab → 从收藏列表中快速找到并置顶 |
| **跨卡片记忆** | 同一个应用在多个门户出现时，⭐ 帮用户记住"我关注过这个" |

### 1.3 高频区规则

- 不区分"高低频区"，不设数量上限
- 卡片固定 6 行高度，超出滚动
- 不需要「在应用中心查看更多」入口（用户自然会滚）

## 2. 默认初始化行为变更

| 场景 | 旧（v0.5） | 新（v0.6） |
|------|-----------|-----------|
| 管理员预设 | 批量收藏 + 批量置顶 | **仅批量置顶**（不初始化收藏） |
| 新用户首次进入 | 收藏一堆（管理员初始化）+ 置顶几个 | 置顶几个（管理员初始化），其余按频率自然排列 |
| 用户取消默认置顶 | 降回频率区 | 降回混排区，继续按频率排 |
| 管理员新增默认置顶 | 对未操作过的用户生效 | 同左，不变 |

> **改的理由：** 管理员强加收藏，用户被迫接受不关心的 ⭐。现在只置顶，尊重度高、卡片更干净。

## 3. 管理弹窗重构

### 3.1 布局变化

**旧（v0.5）：**
```
┌─ 管理我的应用 ──────────────────────────┐
│ 🔍 搜索                                  │
│ 行数设置 / 展示模式                       │
│ 📌 置顶区（可拖拽）                       │
│ 📊 频率区（频率降序）                      │
│ 🔲 补齐区（频率降序补位）                  │
│                             [完成]        │
└──────────────────────────────────────────┘
```

**新（v0.6）：**
```
┌─ 管理我的应用 ──────────────────────────┐
│ 🔍 搜索                                  │
│ ⚠️ 规则说明：卡片展示置顶 + 高频；       │
│    您可以收藏的进行置顶，也可以从常用中直接置顶 │
│ 行数设置 / 展示模式                       │
│ [我的收藏]  [常用应用]  ← 两个tab        │
│ ┌────────────────────────────────────┐   │
│ │ 📌 置顶区（可拖拽调整顺序）         │   │
│ │ ⋮⋮ OA · 平台           [取消📌]    │   │
│ │ 📊 混排区（按频率降序，不可拖拽）    │   │
│ │    人资平台 freq:89      [📌置顶]   │   │
│ │    需求提交 freq:67      [📌置顶]   │   │
│ └────────────────────────────────────┘   │
│                             [完成]        │
└──────────────────────────────────────────┘
```

### 3.2 两个 tab 的差异

| tab | 展示内容 | 排序 | 用途 |
|-----|---------|------|------|
| **我的收藏** | 仅展示已收藏 ⭐ 的应用 | 按访问频率降序 | 快速找到标记过的应用来置顶 |
| **常用应用** | 展示全部权限可见应用 | 按访问频率降序 | 浏览全部，从中直接置顶 |

- 两个 tab 都支持搜索，搜索结果按频率降序
- 两个 tab 内的置顶行为一致：点 📌 → 放到置顶区第一位
- 去掉「+ 添加」按钮（管理弹窗已包含所有操作）

## 4. 应用中心变更

### 4.1 全部应用 — 增加「直接置顶」

| 操作 | 旧（v0.5） | 新（v0.6） |
|------|-----------|-----------|
| 收藏 ⭐ | ✅ 有 | ✅ 有 |
| 置顶 📌 | ❌ 无（需先收藏再回管理弹窗置顶） | ✅ 有 |
| 置顶行为 | — | 置顶后默认放到置顶区第一位，同步卡片立即生效 |

- hover 浮层也增加 📌 按钮
- 用户可在全部应用中一站式完成收藏和置顶，无需绕路

### 4.2 我的应用卡片 — 与门户统一

- 应用中心的「我的应用」卡片内容/样式/行为**完全等于**门户的「我的应用」卡片
- 同一个两区排序模型
- 同一个管理弹窗
- 同一个 hover 详情
