## 详细记录

### 1. 类别名称调整

- **原**：平台 / 功能 / 流程
- **新**：平台 / **工具** / 流程

### 2. 全部应用布局

- **原**：tab 切换后按业务（人资/产研/研发）瀑布流分组
- **新**：tab 切换后直接平铺网格

### 3. 我的应用卡片组成

当前只有「已收藏」。评审后需要包含三类：

| 组成 | 来源 | 说明 |
|------|------|------|
| 已收藏 | 用户主动 ☆ | 现有逻辑 |
| 常用 | 系统统计访问频率 | 未收藏但高频使用 |
| 内置推荐 | 角色默认配置 | 未收藏、未高频，但系统根据角色推荐 |

### 4. 内置推荐的特殊处理

- 需要区别于用户主动收藏的卡片（视觉差异？标识？）
- 用户可以一键转为收藏，也可以忽略
- 内置推荐列表由管理员或角色配置决定

### 5. 「订阅」模型

**当前模型（收藏模型）**：用户主动 ☆ 收藏 → 出现在我的应用。各页面独立维护 personalFavorites。

**目标模型（订阅模型）**：

- 每个角色（HR/研发/等）有默认的「订阅」应用列表
- 用户首次进入该角色页面时，内置默认已订阅的应用
- 用户可以在此基础上增加/移除
- 集团门户作为「汇总视图」：可以看到所有角色页面用户自定义的顺序，作为一个「我的排序」类别

```
集团门户（汇总）
├── 平台类
├── 工具类
├── 流程类
└── 我的排序（回收各角色页面用户的个人顺序）
     ├── 人资门户顺序
     ├── 研发门户顺序
     └── ...
```

### 6. 卡片互通规则

| 页面 | 卡片数据源 | 互通性 |
|------|----------|--------|
| 集团门户 | 全量应用 + 汇总各角色排序 | 拥有全部 |
| HR 角色门户 | HR 范围内的订阅+收藏 | 仅自己 |
| 研发角色门户 | 研发范围内的订阅+收藏 | 仅自己 |
| 应用中心 | 全量应用（浏览+收藏） | 与集团门户共享收藏 |

## 新增：服务台（2026-06-24 补充）

### 概念

**服务台** = 面向全员的 IT 服务角色页面，独立于业务角色（HR/研发等）。

| 维度 | 说明 |
|------|------|
| 定位 | IT 服务统一入口，面向全公司 |
| 典型场景 | 下载 iJomoo 二维码、IT 报修、账号申请等 |
| 形式 | 可在页面右下角预留**小窗口入口**，点击展开服务台面板 |
| 角色 | 作为独立的「服务台」角色页面，全员可见 |

### 页面布局预留

- 门户页面右下角预留服务台小窗口（类似在线客服图标）
- 点击展开 → 展示服务台快捷服务列表
- 同时作为角色页面：可切换进入服务台完整页面
