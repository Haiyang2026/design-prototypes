# 我的应用 — 产品需求文档

> 版本：v0.5 | 日期：2026-07-10 | 状态：补遗漏点版
> 
> 本文档基于 v0.2 复制，按 JMBOS UI/UE 设计系统规范补充了页面范围清单、交互矩阵、组件关键词映射。

## 1. 概述

### 1.1 名词定义

| 中文 | 英文 | 说明 |
|------|------|------|
| 我的应用 | My Apps | 门户页面中的大卡片区域，展示用户已关注的应用/流程 |
| 应用中心 | Apps Center | 全屏弹窗打开，统一管理关注和浏览全部应用 |
| 大卡片 | Big Card | 我的应用这片区域（含标题、按钮、小卡片网格） |
| 小卡片 | Card Item | 大卡片内部的单个应用/流程模块 |
| 收藏 | Favorite | 将应用加入关注列表，从补齐区进入频率区 |
| 置顶 | Pin | 手动将应用固定到置顶区，不受频率排序影响 |
| 置顶区 | Pinned Zone | 卡片第一组：置顶应用区域，用户拖拽排序，不限数量 |
| 频率区 | Frequency Zone | 卡片第二组：已收藏 + 默认模板中未置顶的应用，按访问频率自动排列 |
| 补齐区 | Fill Zone | 卡片第三组：未收藏且不在默认模板中的应用，按频率补位 |
| 默认模板 | Default Template | 管理员为全员预设的初始化配置：提前批量收藏（进频率区）+ 批量置顶（进置顶区）。用户可按需取消 |

### 1.2 背景

集团门户提供统一的应用卡片组件，其他业务团队可「订阅」（复用门户数据）或「共享」（接入自有数据）。本版本聚焦集团门户自身的卡片闭环，后续再扩展多团队接入场景。

### 1.3 涉及页面

- 集团门户 —— 工作台首页，展示「我的应用」卡片
- 应用中心 —— 全屏弹窗打开（前端展示形态为弹窗；开发实现为单独页面或嵌入式组件，待技术讨论确认）

### 1.4 核心逻辑

- **收藏**：员工点击 ☆ 收藏感兴趣的应用。收藏的应用进入频率区（收藏 + 默认模板），按使用频率自动排位。
- **置顶**：员工手动 📌 置顶的应用进入置顶区，按用户设置的固定顺序排列。取消置顶后降回频率区，按频率重新自动排位。取消收藏时自动取消置顶。
- **三区排序模型**：

  | 区域 | 内容 | 排序规则 |
  |------|------|---------|
  | 📌 置顶区 | 置顶的应用（收藏中置顶的 + 默认置顶的） | 用户拖拽固定顺序，不限数量 |
  | 📊 频率区 | 收藏未置顶的 + 默认模板中未置顶的 | 按使用频率降序自动排，不可拖拽 |
  | 🔲 补齐区 | 未收藏且不在默认模板中的应用 | 按使用频率降序自动排，不可拖拽，仅在展示位不足时补位 |

- **默认模板（初始化配置）**：管理员为全员预设的初始化配置 = 提前批量收藏（进频率区）+ 批量置顶（进置顶区）。用户首次进入时卡片已有内容，可按需取消。用户操作后个人自定义优先。

### 1.5 页面范围清单

| # | 页面名称 | 页面类型 | 页面用途 | 入口 |
|---|---------|---------|---------|------|
| P1 | 集团门户 | 门户引擎配置页面 | 工作台首页，展示「我的应用」卡片 | 登录默认进入 |
| P2 | 应用中心 | 待定（全屏弹窗/独立页面） | 统一管理收藏和浏览全部应用 | 门户「我的应用 >」/ 头部菜单 |

> 本版本聚焦集团门户单卡片闭环。应用中心前端展示形态为全屏弹窗，开发实现方式（单独页面或嵌入式组件）待技术讨论确认。

## 2. 用户旅程

### 2.1 首次使用（新员工）

```
打开工作台集团门户页面 → 无收藏、无默认模板
          ↓
我的应用卡片按应用中心返回顺序显示全部应用（均处于补齐区）
          ↓
用户点击「⚙ 管理」→ 管理弹窗中浏览三区列表
  ┌── 置顶区为空（暂无置顶）
  ├── 频率区为空（暂无收藏，未配置默认模板）
  └── 补齐区：全部应用按频率降序排列
          ↓
点击「+ 添加」→ 打开应用池弹窗 → 勾选应用加入频率区
```

> 管理员配置默认模板后，新用户首次进入时相当于管理员已替全员完成了批量收藏和批量置顶。

### 2.2 日常操作

| 操作 | 入口 | 行为 |
|------|------|------|
| 📌 置顶应用 | 管理弹窗 / 卡片 hover | 从频率区或补齐区提升到置顶区末尾 |
| 取消置顶 | 管理弹窗 / 卡片 hover | 降回频率区，按访问频率自动排位 |
| 收藏新应用 | 点击「+ 添加」 | 弹窗中勾选，应用从补齐区进入频率区 |
| 取消收藏 | 管理弹窗 / 卡片 hover | 应用从频率区移除，回到补齐区按频率排位（同时自动取消置顶） |
| 调整置顶顺序 | 管理弹窗拖拽 | 仅置顶区可拖拽，非搜索状态下调换顺序 |
| 查看详情 | 卡片 hover | 展示图标/名称/简介/联系人/📌置顶/☆收藏按钮 |
| 跳转应用 | 点击卡片 | 打开对应应用 URL |
| 发现更多 | 点击「我的应用 >」 | 打开应用中心 |

> 频率区和补齐区**不可拖拽**，按访问频率自动排列。用户只能在非搜索状态下拖拽置顶区。

### 2.3 应用中心跳转路径

- 点击工作台集团门户页面「我的应用 >」→ 全屏弹窗打开应用中心

### 2.4 交互矩阵（页面间交互关系）

| # | 触发页面 | 操作/动作 | 目标页面/弹窗 | 交互类型 | 参数传递 |
|---|---------|----------|-------------|---------|---------|
| 1 | 工作台集团门户页面 | 点击「我的应用 >」 | 应用中心 | 弹窗全屏打开 | 无（应用中心独立加载） |
| 2 | 工作台集团门户页面 | 点击「⚙ 管理」 | 管理弹窗 | 弹窗打开 | 无 |
| 3 | 工作台集团门户页面 | 点击「+ 添加」 | 添加收藏弹窗 | 弹窗打开 | 无 |
| 4 | 工作台集团门户页面 / 应用中心 | 点击小卡片 | 应用/流程目标页面 | 页面跳转 | 应用ID（URL参数） |
| 5 | 工作台集团门户页面 / 应用中心 | 卡片 hover → 「⋯」 | 详情浮层 | 弹窗（浮层） | 应用ID（数据直接渲染） |
| 6 | 应用中心 | 全部应用卡片点击 ☆/★ | 切换收藏状态（加入/移出频率区） | 页内刷新 | 无（前端更新 personalFavorites） |

| 8 | 管理弹窗 | 拖拽置顶区列表项 | — | 弹窗内重排 | 无（前端更新 pinnedApps 顺序） |
| 9 | 管理弹窗 | 点击 ☆ 取消收藏 | — | 弹窗内刷新 | 无（前端更新 personalFavorites + pinnedApps） |

> 交互类型定义：弹窗全屏打开 / 弹窗打开 / 浮层展开 / 页内刷新。共 9 条交互。

## 3. 页面详细设计

### 3.1 大卡片 — 我的应用

#### 3.1.1 布局

```
※ 默认模式无类型页签行，大卡片顶部只有标题和按钮
```

#### 3.1.2 卡片展示

- 每行 6 个卡片
- 支持 1/2/3 行展示（用户可设置）
- 卡片内容：图标 + 名称 + 类型标签（平台/功能/流程）
- 点击整卡 = 跳转应用
- 支持 hover 详情（见 3.3 节）

**置顶超行自动调整：**
- 若置顶区应用数量超出当前行数设置的展示上限，系统自动将行数调整为能容纳所有置顶应用的最小档位
- 调整时弹出提示："您的置顶应用较多（X 个），展示行数已自动调整为 Y 行。可在管理弹窗中手动调整。"
- 用户后续手动调整行数后，不再自动干预

**空状态提示：**
- 置顶区为空时显示："置顶的应用会固定在这里，点击 📌 即可置顶"
- 频率区为空时显示："收藏的应用和默认模板会出现在这里，点击 ☆ 即可收藏，或联系管理员配置默认模板"

#### 3.1.3 类型页签

管理弹窗中提供「展示模式」切换：

| 模式 | 说明 |
|------|------|
| **默认**（简化模式） | 纯网格展示，无类型页签，应用与流程混排。这是卡片的核心简洁体验，降低用户认知负担 |
| **显示分类** | 卡片顶部显示可点击的页签：「全部 / 平台 / 功能 / 流程」|

**交互说明：**
- 显示分类模式下，页签默认选中「全部」，展示所有收藏应用
- 点击页签过滤对应类型的应用
- 如果某类型下没有已收藏的应用，该页签不显示
- 数据来源：平台、功能来源于应用中心接口；流程来源于流程中心接口
- 两种数据在前端合并后按类型分组

#### 3.1.4 按钮

| 按钮 | 功能 |
|------|------|
| 标题「我的应用 >」 | 跳转应用中心 |
| ⚙ 管理 | 打开管理弹窗：三区排序（置顶区/频率区/补齐区）+ 行数设置 + 搜索 |
| + 添加 | 打开「添加收藏」弹窗：全部应用展示，已收藏预选，未收藏置灰可一键收藏 |

#### 3.1.5 新员工场景

新员工首次使用时无收藏无置顶，卡片按应用中心返回顺序展示全部应用（均处于补齐区）。管理员配置默认模板后，相当于帮全员提前做好了批量收藏和批量置顶，用户进来时卡片已有内容。员工主动收藏和置顶后，个人自定义优先于初始化配置。

#### 3.1.6 应用与流程混排规则

卡片默认混排（应用与流程在同一网格中不区分）。显示分类模式下顶部加类型页签（全部/平台/功能/流程）供用户过滤。三区排序规则不受类型影响。

**交互说明：**

**搜索：**
- 顶部搜索框按名称过滤，三区列表中匹配的应用保留显示
- 搜索结果中可操作：📌置顶 / ☆收藏 / ★取消收藏
- **搜索结果中不可拖拽排序**（拖拽仅在非搜索状态下可用）

**置顶区：**
- 非搜索模式下：显示 ⋮⋮ 拖拽手柄，可拖拽调整顺序（`pinnedApps` 数组重排）
- [取消📌]：应用降回频率区，按访问频率重新参与自动排位
- 从频率区或补齐区点 [📌置顶]：应用提升到置顶区**末尾**

**频率区 · 收藏 + 默认模板：**
- 展示已收藏 + 默认模板中未置顶的应用（去重）
- 按访问频率降序自动排列，**不可拖拽**
- [📌置顶]：提升到置顶区末尾
- [★取消收藏]：从频率区移除，回到补齐区按频率排位（若该应用不在默认模板中）

**补齐区 · 其他应用：**
- 未收藏且不在默认模板中的应用，按访问频率降序自动排列
- [📌置顶]：提升到置顶区末尾（同时自动收藏，进入频率区）
- [☆收藏]：加入收藏，进入频率区按频率排位

**显示设置：**
- 展示行数：1行/2行/3行，即时预览
- 展示模式：隐藏分类（默认，小卡片混排不分类型）/ 显示分类（顶部显示分类页签）

**恢复默认：**
- 管理弹窗底部提供「恢复默认」按钮（置灰文字链接样式，不喧宾夺主）
- 点击后二次确认："将恢复为管理员预设的默认配置，您的个人收藏和置顶将被覆盖。确定恢复？"
- 确认后：personalFavorites 和 pinnedApps 重置为管理员的初始化配置

### 3.5 应用中心

应用中心页面与门户大卡片内的小卡片保持一致：同一种小卡片样式、同一种 hover 详情、同一种点击行为。

#### 3.5.1 我的应用

- 展示用户"我的应用"卡片内容，与工作台集团门户页面保持一致
- 遵循三区模型：置顶区（可拖拽）+ 频率区（频率降序）+ 补齐区（频率降序补位）
- 「⚙ 管理」按钮位于卡片右上角，管理弹窗结构同 3.4 节
- 与门户卡片完全一致（同一个小卡片样式、hover 详情、点击行为、头部布局和按钮）

#### 3.5.2 全部应用

- 展示用户权限可见的全部应用目录
- 小卡片样式、hover 详情与门户大卡片内的完全一致
- 每个卡片右下角 ☆/★ 收藏切换
- 已收藏（☆实心）：应用已在频率区 → 点击取消收藏，移出频率区
- 未收藏（☆空心）：应用在补齐区 → 点击收藏，进入频率区
- 卡片点击 = 跳转应用
- hover 浮层不显示📌（需先收藏、进入频率区后才能在管理弹窗或卡片 hover 中置顶）

## 4. 接口需求

### 4.1 数据来源说明

卡片数据来源于两个接口，前端合并展示：

| 数据类型 | 接口来源 | 分类 |
|---------|---------|------|
| 平台、功能 | 应用中心接口 | 按"应用"返回 |
| 流程 | 流程中心接口 | 按"流程"返回 |

两个接口返回的字段格式需要统一，方便前端一次渲染。

### 4.2 应用/流程统一接口格式

应用中心（平台/功能）和流程中心需统一字段格式，方便前端一次渲染。具体字段和接口设计由技术团队确定。

> 以下为业务侧关注的必要字段，供技术团队参考。非技术规范，不可直接用于开发。

| 字段 | 必填 | 在哪里用 | 说明 |
|------|------|---------|------|
| 应用ID | ✅ 必填 | 收藏操作 | 唯一标识，用于收藏/取消收藏时传给接口 |
| 名称 | ✅ 必填 | 小卡片展示 | 应用/流程名称 |
| 类型 | ✅ 必填 | 显示分类页签 | 枚举：平台/功能/流程 |
| 图标 | ✅ 必填 | 小卡片展示 | 图标 URL 或 Unicode |
| 跳转链接 | ✅ 必填 | 小卡片点击 | 点击跳转的 URL |
| 访问频率 | ✅ 必填 | 应用中心排序 | 数字，当前端排序用 |
| 业务分类 | ✅ 必填 | 小卡片标签 + Hover 详情 | '人资'/'产研'/'研发'等 |
| 简介 | ✅ 必填 | Hover 详情 | 短描述文本 |
| 联系人 | ✅ 必填 | Hover 详情 | 联系人列表（姓名/头像/聊天链接），支持点击唤起聊天 |
| 流程主页链接 | ✅ 流程必填 | Hover 详情 | 流程「查看」按钮跳转地址 |
| 流程创建链接 | ✅ 流程必填 | Hover 详情 | 流程「创建」按钮跳转地址 |

> 说明：应用中心（平台/功能）和流程中心的接口建议统一字段格式，方便前端一次渲染。

### 4.3 排序逻辑（三区模型）

#### 4.3.1 三区结构

```
卡片展示顺序（自上而下）：

┌─────────────────────────────────────────────┐
│ 📌 置顶区（人定的）                         │
│ · 默认置顶（管理员预设📌）                  │
│ · 个人置顶（用户手动📌）                    │
│ · 用户拖拽排序，不限制数量                  │
├─────────────────────────────────────────────┤
│ 📊 频率区（数据定的）                       │
│ · 已收藏但未置顶的                          │
│ · 默认模板中未置顶的                        │
│ · 按访问频率降序自动排列，不可拖拽          │
├─────────────────────────────────────────────┤
│ 🔲 补齐区（补位用的）                       │
│ · 既没收藏、也不在默认模板中的              │
│ · 按访问频率降序自动排列，不可拖拽          │
│ · 仅在置顶区+频率区不足展示行数时补位      │
│ · 补位数量 = 展示上限 − 置顶区数量 − 频率区数量，每次渲染动态计算 │
└─────────────────────────────────────────────┘
```

**来源无关原则**：频率区和补齐区内，模板/收藏来源不一的应用混在一起，直接比访问频率数字，无需按来源分组。常用的自然频率高排前面，收藏了但很少用的频率低排后面。

#### 4.3.2 收藏与置顶的关系

- **收藏 ≠ 置顶**：二者解耦
  - 收藏 = 应用从补齐区升入频率区（"关注"）
  - 置顶 = 应用从频率区升入置顶区（"固定顶部"）
- 收藏了但使用频率低 → 仍在频率区靠后位置
- 取消收藏 → 自动取消置顶，应用掉出卡片
- 管理员预设的默认置顶 → 用户也可取消

#### 4.3.3 各页面排序规则

| 页面 | 排序规则 |
|------|---------|
| 门户首页卡片 | 三区模型：置顶区（手动拖拽）+ 频率区（频率降序）+ 补齐区（频率降序补位） |
| 应用中心-我的应用 | 与工作台集团门户页面一致，三区模型 |
| 应用中心-全部应用 | 按接口返回顺序（后端按权限+业务规则排列），前端不额外排序 |
| 管理弹窗 | 置顶区（可拖拽）+ 频率区（频率降序，不可拖拽） |

#### 4.3.5 新用户 vs 老用户

| 用户状态 | 生效逻辑 |
|----------|---------|
| **新用户**（无访问数据） | 置顶区（管理员批量置顶）+ 频率区（管理员批量收藏）+ 补齐区 |
| **老用户**（有访问数据） | 置顶区（管理员批量置顶 + 个人置顶）+ 频率区（按频率降序，管理员批量收藏 + 个人收藏混排）+ 补齐区 |

### 4.4 默认模板（初始化配置）

管理员为全员预设初始化配置，相当于提前完成以下操作：

| 管理员操作 | 效果 | 用户可 |
|-----------|------|--------|
| **批量收藏** | 指定应用进入频率区 | 取消收藏 → 回到补齐区 |
| **批量置顶** | 指定应用进入置顶区（需先被批量收藏） | 取消���顶 → 降回频率区 |

> 简单理解：管理员替全员提前做了一次批量收藏 + 批量置顶。用户看到的就像管理员已经帮他操作过了。

初始化时机：

| 场景 | 效果 |
|------|------|
| 新用户首次进入 | 直接加载管理员的初始化配置（批量收藏 + 批量置顶） |
| 老用户从未操作过卡片 | 按最新初始化配置更新 |
| 老用户已操作过（收藏/置顶/拖拽） | **不受初始化配置变更影响**（个人自定义优先） |

#### 4.4.1 模板中应用的来源

初始化配置中的应用必须来自用户权限范围内的应用池。管理员只能从用户有权限访问的应用中选择。

### 4.5 访问频率数据

访问频率由应用中心、流程中心各自提供现成的数据。前端直接取接口返回的结果即可，无需我们设计方案。
