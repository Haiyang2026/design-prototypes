## 详细说明

### 27-1 菜单顺序和收藏自己存

见 `顶部菜单-平台自存数据需求.md` §2：
- 菜单项顺序 → 用户拖拽后自存，恢复默认时覆盖
- 菜单收藏列表 → 个人标记，不参与排序
- 默认门户页ID → 默认全员集团门户，后台可配

---

### 27-2 菜单源头删除处理

- 上游数据源删除某菜单项后，我方接口拉不到该条目
- 前端不展示该菜单项（同时清理自存数据中的该ID）
- 与夏龙沟通一致

---

### 27-3 页面自定义中台可行性

- 开发已评估，中台可以实现
- 无需额外处理

---

### 27-4 ijomoo二维码直接漏出来

**顶栏改动：**

```
[JOMO] [集团门户] [研发门户] … [📱] [👤⚙]
```

- 📱 图标放在顶栏右侧，头像左边
- 点击或悬浮时弹出二维码浮层（ijomoo 下载二维码）
- 二维码直接可见，不藏到抽屉里

**⚠️ 关联改动：去掉旧工作台页面上的悬浮按钮**
- 旧版工作台页面上若有 ijomoo 下载悬浮按钮
- 开发需一并移除
- 海洋上线后验证

---

### 27-5 常用语言/皮肤直接选

**个人设置抽屉改动：**

```
┌─ 个人设置 ───────────────────────┐
│  👤 头像 / 姓名 / 部门            │
│                                   │
│  📋 页面设置            →        │
│                                   │
│  🌐 常用语言                      │
│  ○ 中文  ○ English  ○ 日本語     │
│                                   │
│  🎨 常用皮肤                      │
│  ○ 浅色  ○ 深色  ○ 跟随系统      │
│                                   │
│          [退出登录]               │
└──────────────────────────────────┘
```

- 语言和皮肤选项直接平铺在抽屉里
- 单选即生效，无需二次点击进入
- 不提顶栏（顶栏保持简洁）

---

### 27-6 默认全员集团门户

**规则：**

| 场景 | 行为 |
|------|------|
| 无后台配置 | 全员默认进入集团门户 |
| 管理员在后台为某用户/角色配置了其他默认页 | 对应账号优先展示配置的默认页 |
| 用户在设置中修改了默认页 | 个人设置优先于后台配置 |

**存储：** 见 `顶部菜单-平台自存数据需求.md` §2.3
