# 应用中心接口与数据需求

> 版本：v0.2 | 日期：2026-07-14 | 状态：已提交中台，期望上线 2026-08-15  
> 禅道：[#13627](https://zentao.jomoo.com/zentaopms/www/story-view-13627-0--story.html) — 【应用中心】页面改造，需要中台提供相应接口  
> 说明：本文档涵盖应用中心、流程中心需提供的数据，以及我们平台自存数据、对外输出的排序结果

---

## 1. 外部需提供的数据

> 对象：应用中心接口 + 流程中心接口  
> 用途：门户「我的应用」卡片 & 应用中心「全部应用」tab 渲染

### 1.1 通用出参（应用中心和流程中心统一）

| 字段 | 必填 | 说明 |
|------|------|------|
| 应用ID | ✅ | 唯一标识 |
| 名称 | ✅ | 应用/流程名称 |
| 图标 | ✅ | 图标 URL |
| 类型 | ✅ | 平台 / 功能 / 流程 |
| 跳转链接 | ✅ | 点击卡片跳转的 URL。应用中心 = 应用跳转链接；流程中心 = 流程主页链接 |
| 创建链接 | 非必填 | 跳转创建页的 URL，无则留空 |
| 业务分类 | ✅ | 人资 / 产研 / 研发等 |
| 简介 | ✅ | 短描述文本（Hover 详情） |
| 联系人 | ✅ | 姓名/头像列表（仅展示，暂不支持唤起聊天） |
| 访问频率 | ✅ | 该用户对此应用的访问频率值（混排区排序用） |

### 1.2 应用中心额外字段

> 应用中心有标签展示：工具/平台 + 对应子类别

| 字段 | 必填 | 说明 |
|------|------|------|
| 主标签 | ✅ | 工具 / 平台 |
| 子类别 | ✅ | 如「人资平台」「产研工具」等 |

### 1.3 接口列表

| 编号 | 接口 | 用途 | 对象 |
|------|------|------|------|
| API-1 | 获取用户权限可见的应用列表 | 卡片渲染数据源 | 应用中心 + 流程中心 |
| API-2 | 获取全部应用（含分类标签） | 应用中心「全部应用」tab | 应用中心 + 流程中心 |

> **排序说明**：接口无需返回排序后的结果，只需返回原始数据 + 访问频率。前端自行按两区模型排序：置顶区（我们自存顺序）+ 混排区（频率降序）。

---

## 2. 我们自己平台要存的数据

> 对象：我方平台前端/后端  
> 用途：用户的置顶顺序 + 收藏列表，不属于外部接口

| 数据 | 说明 |
|------|------|
| 用户ID | — |
| 置顶应用ID列表 | 数组，按用户拖拽后的顺序排列 |
| 收藏应用ID列表 | 数组，用户收藏的应用ID |

> 置顶和收藏完全独立：取消收藏不影响置顶，取消置顶不影响收藏。

---

## 3. 我们对外输出的排序结果

> 对象：流程中心 / 应用侧需要获取用户应用排序时调用  
> 用途：将我方两区排序结果（置顶 + 频率）返回给调用方

**入参：**

| 参数 | 必填 | 说明 |
|------|------|------|
| 用户ID | ✅ | — |

**出参：**

| 字段 | 说明 |
|------|------|
| 应用ID列表 | 按最终展示顺序排列：置顶区（用户顺序）+ 混排区（频率降序） |

---

## 4. 约束说明

- **上游删除**：数据源删除某应用后，§1 接口不再返回，我方不展示（含我的应用卡片 + 全部应用）。
- **无logo兜底**：我方自行处理（怀伟出 4 个通用默认图标），接口侧无需额外处理。
- **频率数据**：由应用中心/流程中心各自提供现成数据，前端直接取用。
- **权限过滤**：接口返回的数据已是用户权限范围内的，前端不做二次过滤。
