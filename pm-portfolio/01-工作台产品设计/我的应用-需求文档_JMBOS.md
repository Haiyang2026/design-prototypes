> 版本：v0.6.1 | 日期：2026-07-15 | 状态：管理弹窗结构修正版
> 
> 本文档基于 v0.2 复制，按 JMBOS UI/UE 设计系统规范补充了页面范围清单、交互矩阵、组件关键词映射。

## 1. 概述

### 1.1 名词定义

| 中文 | 英文 | 说明 |
|------|------|------|
| 我的应用 | My Apps | 门户页面中的大卡片区域，展示用户已关注的应用/流程 |
| 应用中心 | Apps Center | 全屏弹窗打开，统一管理关注和浏览全部应用 |
| 大卡片 | Big Card | 我的应用这片区域（含标题、按钮、小卡片网格） |
| 小卡片 | Card Item | 大卡片内部的单个应用/流程模块 |
| 收藏 | Favorite | ⭐ 标记应用为个人关注，不参与排序；可作为置顶的快捷入口 |
| 置顶 | Pin | 手动将应用固定到置顶区，不受频率排序影响 |
| 置顶区 | Pinned Zone | 卡片最前面区域：置顶应用按用户指定顺序排列，不限数量 |
| 混排区 | Main Zone | 卡片后面区域：用户权限可见的全部应用，按访问频率降序自动排列，不可拖拽 |
| 默认置顶 | Default Pin | 管理员为全员预设的初始化置顶，用户可按需取消 |

### 1.2 背景

集团门户提供统一的应用卡片组件，其他业务团队可「订阅」（复用门户数据）或「共享」（接入自有数据）。本版本聚焦集团门户自身的卡片闭环，后续再扩展多团队接入场景。

### 1.3 涉及页面

- 集团门户 —— 工作台首页，展示「我的应用」卡片
- 应用中心 —— 全屏弹窗打开（前端展示形态为弹窗；开发实现为单独页面或嵌入式组件，待技术讨论确认）

### 1.4 核心逻辑

- **收藏**：员工点击 ⭐ 标记应用为个人关注。收藏不参与排序，不影响卡片展示顺序；收藏作为置顶的快捷入口（管理弹窗「我的收藏」tab）和身份标记（卡片 ⭐ 角标）。
- **置顶**：员工手动 📌 置顶的应用进入置顶区，按用户设置的固定顺序排列。取消置顶后降回混排区，按频率重新自动排位。
- **两区排序模型**：

  | 区域 | 内容 | 排序规则 |
  |------|------|---------|
  | 📌 置顶区 | 管理员默认置顶 + 用户手动置顶 | 用户拖拽固定顺序，不限制数量 |
  | 📊 混排区 | 用户权限可见的全部应用 | 按访问频率降序自动排，不可拖拽 |

- **默认置顶（初始化配置）**：管理员为全员预设的初始化配置 = 批量置顶（进置顶区）。用户首次进入时卡片已有置顶内容，可按需取消。用户操作后个人自定义优先。
- **来源无关原则**：混排区内所有应用直接比访问频率数字，无需按来源分组。常用的自然频率高排前面。

### 1.5 页面范围清单

| # | 页面名称 | 页面类型 | 页面用途 | 入口 |
|---|---------|---------|---------|------|
| P1 | 集团门户 | 门户引擎配置页面 | 工作台首页，展示「我的应用」卡片 | 登录默认进入 |
| P2 | 应用中心 | 待定（全屏弹窗/独立页面） | 统一管理收藏和浏览全部应用 | 门户「我的应用 >」/ 头部菜单 |

> 本版本聚焦集团门户单卡片闭环。应用中心前端展示形态为全屏弹窗，开发实现方式（单独页面或嵌入式组件）待技术讨论确认。

## 2. 用户旅程

### 2.1 首次使用（新员工）

```
打开工作台集团门户页面
          ↓
我的应用卡片展示：
  ┌── 置顶区：管理员默认置顶的应用
  └── 混排区：权限可见的全部应用（按频率降序）
          ↓
点击「⚙ 管理」→ 管理弹窗：
  [我的收藏] tab → 空（暂无收藏）
  [常用应用] tab → 全部应用，可搜索、置顶
```

> 管理员配置默认置顶后，新用户首次进入时卡片已有置顶内容。

### 2.2 日常操作

| 操作 | 入口 | 行为 |
|------|------|------|
| 📌 置顶应用 | 管理弹窗 / 应用中心 / 卡片 hover | 加入置顶区第一位（后续可拖拽调整） |
| 取消置顶 | 管理弹窗 / 卡片 hover | 降回混排区，按访问频率自动排位 |
| 收藏 ⭐ | 卡片 hover / 应用中心 | 标记为个人关注，带 ⭐ 角标，不影响排序 |
| 取消收藏 | 卡片 hover / 应用中心 | 移除 ⭐ 标记 |
| 调整置顶顺序 | 管理弹窗拖拽 | 仅置顶区可拖拽，非搜索状态下调换顺序 |
| 查看详情 | 卡片 hover | 展示图标/名称/简介/联系人/📌置顶/⭐收藏 |
| 跳转应用 | 点击卡片 | 打开对应应用 URL |
| 发现更多 | 点击「我的应用 >」 | 打开应用中心 |

> 混排区**不可拖拽**，按访问频率自动排列。

### 2.3 应用中心跳转路径

- 点击工作台集团门户页面「我的应用 >」→ 全屏弹窗打开应用中心

### 2.4 交互矩阵（页面间交互关系）

| # | 触发页面 | 操作/动作 | 目标页面/弹窗 | 交互类型 | 参数传递 |
|---|---------|----------|-------------|---------|---------|
| 1 | 工作台集团门户页面 | 点击「我的应用 >」 | 应用中心 | 弹窗全屏打开 | 无（应用中心独立加载） |
| 2 | 工作台集团门户页面 | 点击「⚙ 管理」 | 管理弹窗 | 弹窗打开 | 无 |
| 3 | 工作台集团门户页面 / 应用中心 | 点击小卡片 | 应用/流程目标页面 | 页面跳转 | 应用ID（URL参数） |
| 4 | 工作台集团门户页面 / 应用中心 | 卡片 hover → 「⋯」 | 详情浮层 | 弹窗（浮层） | 应用ID（数据直接渲染） |
| 5 | 应用中心-全部应用 | 卡片 ☆/⭐ 切换收藏 | — | 页内刷新 | 无（前端更新收藏状态） |
| 6 | 应用中心-全部应用 | 卡片 📌 置顶 | — | 页内刷新 | 无（前端更新 pinnedApps，置顶区第一位） |
| 7 | 管理弹窗 | 📌 置顶 / 取消置顶 | — | 弹窗内刷新 | 无（前端更新置顶列表） |
| 8 | 管理弹窗 | 拖拽置顶区列表项 | — | 弹窗内重排 | 无（前端更新 pinnedApps 顺序） |
| 9 | 管理弹窗 | 切换 tab（我的收藏/常用应用） | — | 弹窗内刷新 | 无（前端切换数据源） |

> 交互类型定义：弹窗全屏打开 / 弹窗打开 / 浮层展开 / 页内刷新。共 9 条交互。

## 3. 页面详细设计

### 3.1 大卡片 — 我的应用

#### 3.1.1 布局

```
┌─────────────────────────────────────────────────────────┐
│ [我的应用 >]        [🔍 搜索应用...]     [⚙ 管理]       │ ── 头部
├─────────────────────────────────────────────────────────┤
│ ◈ 显示分类模式（可选）：                                │
│ [全部] [平台] [功能] [流程]                            │ ← 可点击过滤
├─────────────────────────────────────────────────────────┤
│ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │
│ │ Icon │ │ Icon │ │ Icon │ │ Icon │ │ Icon │ │ Icon │ │
│ │ 名称  │ │ 名称  │ │ 名称  │ │ 名称  │ │ 名称  │ │ 名称  │ │
│ │ 标签  │ │ 标签  │ │ 标签  │ │ 标签  │ │ 标签  │ │ 标签  │ │
│ └──────┘ └──────┘ └──────┘ └──────┘ └──────┘ └──────┘ │
│ ┌──────┐ ┌──────┐ ┌──────┐                             │
│ │ Icon │ │ Icon │ │ Icon │                             │
│ │ 名称  │ │ 名称  │ │ 名称  │                             │
│ │ 标签  │ │ 标签  │ │ 标签  │                             │
│ └──────┘ └──────┘ └──────┘                             │
└─────────────────────────────────────────────────────────┘

▲ 上方整个区域是「大卡片」，每个方块是一个「小卡片」
※ 默认模式无类型页签行，大卡片顶部只有标题和按钮
```

#### 3.1.2 卡片展示

- 每行 6 个卡片
- 支持 1/2/3 行展示（用户可设置），固定 6 行高度，超出滚动
- 卡片内容：图标 + 名称 + 类型标签（平台/功能/流程）
- 收藏的应用带 ⭐ 角标
- 点击整卡 = 跳转应用
- 支持 hover 详情（见 3.3 节）

**置顶超行自动调整：**
- 若置顶区应用数量超出当前行数设置的展示上限，系统自动将行数调整为能容纳所有置顶应用的最小档位
- 调整时弹出提示："您的置顶应用较多（X 个），展示行数已自动调整为 Y 行。可在管理弹窗中手动调整。"
- 用户后续手动调整行数后，不再自动干预

**空状态提示：**
- 置顶区为空时显示："置顶的应用会固定在这里，点击 📌 即可置顶"
- 混排区为空时显示："暂无权限可见的应用，请联系管理员"

#### 3.1.3 类型页签

管理弹窗中提供「展示模式」切换：

| 模式 | 说明 |
|------|------|
| **默认**（简化模式） | 纯网格展示，无类型页签，应用与流程混排。这是卡片的核心简洁体验，降低用户认知负担 |
| **显示分类** | 卡片顶部显示可点击的页签：「全部 / 平台 / 功能 / 流程」|

**交互说明：**
- 显示分类模式下，页签默认选中「全部」，展示所有收藏应用
- 点击页签过滤对应类型的应用
- 如果某类型下没有已收藏的应用，该页签不显示
- 数据来源：平台、功能来源于应用中心接口；流程来源于流程中心接口
- 两种数据在前端合并后按类型分组

#### 3.1.4 按钮

| 按钮 | 功能 |
|------|------|
| 标题「我的应用 >」 | 跳转应用中心 |
| 🔍 搜索 | 按名称实时过滤卡片中的应用 |
| ⚙ 管理 | 打开管理弹窗：两个tab（我的收藏/常用应用）切换 + 置顶区拖拽 + 行数设置 + 搜索 |

#### 3.1.5 新员工场景

新员工首次使用时无收藏无置顶，卡片展示全部权限可见应用（按频率降序）。管理员配置默认置顶后，新用户首次进入时卡片已有置顶内容。员工手动置顶后，个人自定义优先于初始化配置。

#### 3.1.6 应用与流程混排规则

卡片默认混排（应用与流程在同一网格中不区分）。显示分类模式下顶部加类型页签（全部/平台/功能/流程）供用户过滤。两区排序规则不受类型影响。

- 「查看」：仅流程类应用，跳转到流程主页
- 联系人列表可点击，唤起聊天

### 3.3 管理弹窗

```
┌──────────────────────────────────────────────────────┐
│ 管理我的应用                                  [✕]   │
├──────────────────────────────────────────────────────┤
│ ⚠️ 卡片显示的是置顶 + 高频应用；                     │
│    您可以选择收藏的应用进行置顶，也可以从常用应用中直接置顶 │
├──────────────────────────────────────────────────────┤
│ [🔍 搜索应用名称...]                                 │ ← 搜索结果中可置顶，不可拖拽
├──────────────────────────────────────────────────────┤
│ 📐 显示设置                                          │
│ 展示行数：○ 1行  ○ 2行  ○ 3行                      │
│                                                    │
│ 展示模式：○ 隐藏分类（默认）                          │
│           ○ 显示分类（按类型页签过滤）         │
├──────────────────────────────────────────────────────┤
│ 📌 置顶区（始终在顶部，跨 tab 固定，可拖拽）         │
│ ⋮⋮ [Icon] OA · 平台 · 人资                [取消📌] │
│ ⋮⋮ [Icon] 人资平台 · 平台 · 人资           [取消📌] │
├──────────────────────────────────────────────────────┤
│ [⭐ 我的收藏]  [📊 常用应用]                        │ ← 两个 tab 切换
│                                                      │
│ （当前 tab 应用列表，按频率降序，不可拖拽）         │
│     [Icon] 入职办理 · 功能 · 人资  freq:89 [📌置顶] │
│     [Icon] 需求提交 · 功能 · 产研  freq:67 [📌置顶] │
│     [Icon] ACS 2.0 · 平台 · 产研       freq:45 [📌置顶] │
├──────────────────────────────────────────────────────┤
│ [恢复默认]                                [完成]    │
└──────────────────────────────────────────────────────┘
```

**交互说明：**

**整体结构：**
- 弹窗纵向三段：①置顶区（顶部固定） ②显示设置 + 搜索 ③+ 添加置顶（带 tab）
- 置顶区跨 tab 固定显示，不随 tab 切换变化
- tab 切换只影响"添加置顶"区的应用列表

**规则说明：**
- 弹窗顶部新增一行说明文字，告知用户卡片展示逻辑：置顶 + 高频。用户可从「我的收藏」tab 快速置顶，也可从「常用应用」tab 直接置顶。

**置顶区（顶部固定）：**
- 显示所有已置顶的应用，按用户拖拽顺序排列
- 非搜索模式下：显示 ⋮⋮ 拖拽手柄，可拖拽调整顺序（`pinnedApps` 数组重排）
- 搜索中：隐藏拖拽手柄
- [取消📌]：应用从置顶区移除，回到「添加置顶」区按 tab 排序
- 跨 tab 固定：无论选哪个 tab，置顶区内容都一样

**+ 添加置顶区：**
- 两个 tab：
  - ⭐ 我的收藏：未置顶的收藏应用，按访问频率降序
  - 📊 常用应用：未置顶的常用应用，按访问频率降序
- 列表项展示 [📌置顶] 按钮：点击后从当前列表移除，进入置顶区**第一位**
- tab 切换：列表数据按对应数据源刷新
- 两个 tab 均支持搜索

**搜索：**
- 顶部搜索框按名称过滤列表（仅过滤"+ 添加置顶"区，置顶区不受影响）
- 搜索结果中可操作：📌置顶
- 搜索中：置顶区拖拽手柄隐藏（不可拖拽）

**显示设置：**
- 展示行数：1行/2行/3行，即时预览
- 展示模式：隐藏分类（默认，小卡片混排不分类型）/ 显示分类（顶部显示分类页签）

**恢复默认：**
- 管理弹窗底部提供「恢复默认」按钮（置灰文字链接样式，不喧宾夺主）
- 点击后二次确认："将恢复为管理员预设的默认配置，您的个人置顶将被覆盖。确定恢复？"
- 确认后：pinnedApps 重置为管理员的初始化配置

### 3.4 应用中心

应用中心页面与门户大卡片内的小卡片保持一致：同一种小卡片样式、同一种 hover 详情、同一种点击行为。

#### 3.4.1 我的应用

- 展示与工作台集团门户页面完全一致的「我的应用」卡片
- 遵循两区模型：置顶区（可拖拽）+ 混排区（频率降序）
- 「⚙ 管理」按钮位于卡片右上角，管理弹窗结构同 3.3 节
- 与门户卡片完全一致（同一个小卡片样式、hover 详情、点击行为、头部布局和按钮）

#### 3.4.2 全部应用

- 展示用户权限可见的全部应用目录
- 顶部🔍搜索框：按名称实时过滤全部应用
- 小卡片样式、hover 详情与门户大卡片内的完全一致
- 每个卡片右下角：⭐ 收藏切换 + 📌 置顶按钮
- 📌 置顶：应用直接放到置顶区第一位，同步「我的应用」卡片立即生效
- ⭐ 收藏：标记为个人关注，带 ⭐ 角标，不影响排序
- 卡片点击 = 跳转应用

## 4. 接口需求

### 4.1 数据来源说明

卡片数据来源于两个接口，前端合并展示：

| 数据类型 | 接口来源 | 分类 |
|---------|---------|------|
| 平台、功能 | 应用中心接口 | 按"应用"返回 |
| 流程 | 流程中心接口 | 按"流程"返回 |

两个接口返回的字段格式需要统一，方便前端一次渲染。

**上游删除处理：** 当上游数据源删除了某个应用/流程，我方拉取数据时拉不到该条目，则不在「我的应用卡片」和「全部应用」中展示该条目（已与夏龙沟通确认）。

### 4.2 应用/流程统一接口格式

应用中心（平台/功能）和流程中心需统一字段格式，方便前端一次渲染。具体字段和接口设计由技术团队确定。

| 名称 | ✅ 必填 | 小卡片展示 | 应用/流程名称 |
| 类型 | ✅ 必填 | 显示分类页签 | 枚举：平台/功能/流程 |
| 图标 | ✅ 必填 | 小卡片展示 | 图标 URL 或 Unicode |
| 跳转链接 | ✅ 必填 | 小卡片点击 | 点击跳转的 URL |

### 4.3 排序逻辑（两区模型）

#### 4.3.1 两区结构

```
卡片展示顺序（自上而下）：

┌─────────────────────────────────────────────┐
│ 📌 置顶区（人定的）                         │
│ · 管理员默认置顶 + 用户手动置顶            │
│ · 用户拖拽排序，不限制数量                  │
├─────────────────────────────────────────────┤
│ 📊 混排区（数据定的）                       │
│ · 用户权限可见的全部应用                    │
│ · 统一按访问频率降序自动排列，不可拖拽      │
│ · 收藏的应用带 ⭐ 角标，不影响排序           │
│ · 卡片固定 6 行 + 滚动，不设数量上限        │
└─────────────────────────────────────────────┘
```

**来源无关原则**：混排区内所有应用直接比访问频率数字，无需按来源分组。常用的自然频率高排前面，收藏了但很少用的频率低排后面。

#### 4.3.2 收藏与置顶的关系

- **收藏 ≠ 置顶**：二者完全独立
  - 收藏 = ⭐ 个人标记，不参与排序
  - 置顶 = 📌 固定到顶部
- 收藏不带来任何排序加成
- 收藏作为置顶的快捷入口：管理弹窗「我的收藏」tab
- 取消收藏不影响置顶状态

#### 4.3.3 各页面排序规则

| 页面 | 排序规则 |
|------|---------|
| 门户首页卡片 | 两区模型：📌置顶区（手动拖拽）+ 📊混排区（频率降序） |
| 应用中心-我的应用 | 与门户完全一致 |
| 应用中心-全部应用 | 按接口返回顺序，前端不排序 |
| 管理弹窗 | 📌置顶区（可拖拽）+ 应用列表（频率降序，不可拖拽） |

#### 4.3.5 新用户 vs 老用户

| 用户状态 | 生效逻辑 |
|----------|---------|
| **新用户**（无访问数据） | 置顶区（管理员默认置顶）+ 混排区（权限可见全量） |
| **老用户**（有访问数据） | 置顶区（管理员默认置顶 + 个人置顶）+ 混排区（按频率降序） |

### 4.4 默认置顶（初始化配置）

管理员为全员预设初始化配置，相当于提前完成批量置顶操作：

| 管理员操作 | 效果 | 用户可 |
|-----------|------|--------|
| **批量置顶** | 指定应用进入置顶区 | 取消置顶 → 降回混排区 |

> 与 v0.5 的区别：管理员不再默认批量收藏。收藏是用户的个人行为，不纳入初始化配置。

初始化时机：

| 场景 | 效果 |
|------|------|
| 新用户首次进入 | 加载管理员的默认置顶配置 |
| 老用户从未操作过置顶 | 按最新默认置顶配置更新 |
| 老用户已操作过（置顶/拖拽） | **不受初始化配置变更影响**（个人自定义优先） |

#### 4.4.1 模板中应用的来源

初始化配置中的应用必须来自用户权限范围内的应用池。管理员只能从用户有权限访问的应用中选择。

### 4.5 访问频率数据

访问频率由应用中心、流程中心各自提供现成的数据。前端直接取接口返回的结果即可，无需我们设计方案。
