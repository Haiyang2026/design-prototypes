#### 0.2 业务背景

工作台顶部菜单当前存在两个已识别的问题，需在 V1.0 版本统一修复：

1. **菜单不可以调整顺序**：当前菜单顺序完全由后台固定，用户无法根据使用习惯调整展示顺序。
2. **角色门户无法设置默认优先展示的门户页面**：例如研发人员登录后应直接看到研发平台，但当前系统不支持此设置。

> 菜单左右留白问题由开发团队统一规划，不在本需求文档中讨论。

| P2 | 个人设置面板 | 右侧抽屉面板 | 展示个人信息、功能入口和退出登录 | 点击顶部右侧个人头像（⚙） |

#### 0.4 核心逻辑

1. **菜单可见性**：用户能看到哪些菜单项，由门户引擎的菜单权限设置决定。
2. **默认顺序与默认页**：菜单项的默认顺序和默认页面来自门户引擎后台菜单设置。
3. **用户自定义**：每个用户都可自定义菜单项顺序和默认页面，保存后生成专属菜单配置。
4. **后台默认设置**：管理员可在后台针对某一组用户设置默认页面。用户个人设置优先于后台组设置。
5. **默认页优先级**：用户个人设置 > 后台用户组设置 > 系统兜底（集团门户）。

### 1. 页面与组件设计

#### 1.1 页面分组

| 分组名称 | 分组描述 | 包含页面/组件 | 入口 |
|----------|----------|---------------|------|
| 顶部导航区 | 工作台顶部横向导航栏 | 顶部菜单（含菜单项、「更多」下拉） | 登录后始终可见 |
| 个人设置面板 | 右侧抽屉面板 | 个人信息区、功能区、退出区 | 顶部右侧个人头像 |
| 页面设置弹窗 | 全屏弹窗 | 菜单设置（左1/3）、页面布局设置（右2/3） | 个人设置面板「页面设置」 |

##### 1.1.1 顶部菜单 — 正常模式

**布局结构**：

顶部菜单横向展示权限内可见的菜单项，每个菜单项为文字标签。当前选中的菜单项以高亮样式显示。

**V1.0 变更点：响应式展示** — 菜单项数量根据屏幕宽度动态计算：屏幕较窄时仅展示少量高频菜单项，超出部分收入右侧「更多」按钮的下拉菜单；屏幕较宽时自动展示更多菜单项。

**交互说明**：
- 点击菜单项跳转到对应门户页面，当前项高亮
- 点击个人头像（右下角带 ⚙ 标识）打开个人设置面板

##### 1.1.2 个人设置面板

**布局结构**：

```
┌─────────────────────────────────┐
│ 个人信息区                        │
│ ├─ 账号（工号）— 只读            │
│ ├─ 办公手机（ⓘ）— 只读，hover 气泡│
│ ├─ 邮箱 — 只读                   │
│ ├─ 个人手机（✏）— 必填，可修改   │
│ ├─ 座机（✏）— 非必填，可修改     │
│ └─ 修改密码 — 确认弹窗后跳转      │
├─────────────────────────────────┤
│ 功能区                           │
│ ├─ 页面设置                      │
│ ├─ 下载移动端                    │
│ ├─ 消息（带数量）                │
│ ├─ 常用语言                      │
│ └─ 常用皮肤                      │
├─────────────────────────────────┤
│ 退出区                           │
│ └─ 退出登录                      │
└─────────────────────────────────┘
```

**交互说明**：

| 元素 | 功能 | 说明 |
|------|------|------|
| 账号 | 只读展示 | 展示工号 |
| 办公手机 | 只读，hover ⓘ 提示 | 气泡展示变更申请流程说明 |
| 邮箱 | 只读展示 | 完整展示 |
| 个人手机 | 必填，行内编辑 | 点击 ✏ → 切换为输入框 + 保存/取消 |
| 座机 | 非必填，行内编辑 | 同上 |
| 修改密码 | 确认弹窗后跳转 | 先全屏确认弹窗，再跳转用户中心 |
| 页面设置 | 打开全屏弹窗 | 进入菜单设置 + 页面布局设置 |
| 下载移动端 | 跳转下载页 | — |
| 消息 | 进入消息中心 | 展示未读消息数量 |
| 常用语言 | 切换语言 | 中文/英文 |
| 常用皮肤 | 切换主题 | 亮色/暗色 |
| 退出登录 | 退出当前账号 | 面板底部单独展示 |

##### 1.1.3 页面设置全屏弹窗

**布局结构**：

弹窗占满整个浏览器视口，底栏固定展示「保存」和「取消」按钮。主体左右分区：

- **左侧「菜单设置」区（占 1/3 宽度）**：菜单设置模式，支持拖拽调整顺序和设置默认页
- **右侧「页面布局设置」区（占 2/3 宽度）**：顶部「菜单选择」下拉框，嵌入门户引擎原布局页

**交互说明**：

| 元素 | 功能 | 说明 |
|------|------|------|
| 左侧菜单设置 | 拖拽排序 + 设置默认页 | 顶部「恢复默认」按钮，恢复后台默认配置 |
| 右侧页面布局 | 选择门户 + 嵌入布局页 | 「菜单选择」仅列出支持布局设置的门户 |
| 全局保存 | 同时保存菜单设置 + 当前布局页 | 无需手动切换菜单 |
| 布局切换防丢 | 有未保存修改时弹出确认 | 保存并切换 / 不保存 / 取消 |
| 关闭/取消防丢 | 有未保存修改时弹出确认 | 保存并退出 / 直接退出 / 取消 |
| 菜单选择范围 | 仅列出支持布局的门户 | V1.0 仅集团门户支持 |

##### 1.1.4 菜单设置模式

**交互说明**：

| 元素 | 功能 | 说明 |
|------|------|------|
| 拖拽手柄 | 拖拽调整菜单项顺序 | 松开后新位置即时生效 |
| 默认页 ★ 星标 | 标记/取消默认页 | 同一时间仅一个菜单项为默认页 |
| 恢复默认 | 恢复后台默认配置 | 点击后二次确认 |
| 保存 | 保存到个人后台，退出设置模式 | 失败时留在弹窗并提示「保存失败，请重试」 |
| 取消 | 放弃修改，退出设置模式 | 若右侧布局有未保存修改，弹窗确认 |

#### 1.2 组件联动关系

##### 1.2.1 默认页优先级联动

```
用户登录
  ↓
检查用户个人设置
  ├─ 有个人默认页 → 展示该页面，对应菜单项高亮
  └─ 无个人默认页 → 检查后台组设置
       ├─ 有组设置 → 展示对应页面
       └─ 无组设置 → 展示集团门户（兜底）
```

##### 1.2.2 菜单设置与顶部菜单联动

```
菜单设置模式中保存
  ↓
顶部菜单即时刷新（顺序 + 默认页生效）
  ↓
内容区切换到默认页（如设置了默认页）
```

##### 1.2.3 权限变更联动

```
后台变更用户菜单权限
  ↓
用户重新登录/刷新 → 菜单项自动同步
  ↓
用户无法在设置模式中增删菜单项，只能调整已有可见项的顺序
```

### 2. 交互流程

#### 2.1 交互步骤总览

| 步骤ID | 步骤名称 | 触发页面 | 目标页面/弹窗 | 交互类型 | 说明 |

| 5 | 拖拽排序 | 菜单设置模式 | 当前页面 | 页内操作 | 拖拽菜单项调整顺序 |
| 6 | 设置默认页 | 菜单设置模式 | 当前页面 | 页内操作 | 点击 ★ 标记默认页 |

| 8 | 取消菜单设置 | 菜单设置模式 | 顶部菜单 | 页内刷新 | 恢复到进入设置前的状态 |
| 9 | 恢复默认 | 菜单设置模式 | 菜单设置模式 | 页内操作 | 二次确认后恢复后台默认配置 |
| 10 | 修改个人信息 | 个人设置面板 | 当前面板 | 行内编辑 | 个人手机/座机切换为输入框 |

#### 2.2 用户旅程

##### 2.2.1 首次使用（无个人设置 — 新员工场景）

```
新员工登录工作台
  ↓
菜单按权限默认顺序展示
  ↓
默认页面按后台用户组设置，未设置则展示集团门户
  ↓
用户可随时进入菜单设置模式自定义
```

##### 2.2.2 自定义菜单

```
用户进入菜单设置模式
  ↓
拖拽调整菜单项顺序
  ↓
点击 ★ 设置某个菜单项为默认页
  ↓
点击保存 → 菜单顺序和默认页生效
  ↓
后续登录自动进入默认页，菜单按自定义顺序展示
```

##### 2.2.3 后台按用户组设置默认页

```
管理员在后台为研发组设置默认页为「研发平台」
  ↓
该组所有无个人设置的用户登录后默认展示研发平台
  ↓
用户可自行更改默认页，个人设置覆盖后台组设置
```

#### 2.3 特殊交互场景

##### 2.3.1 仅有一个菜单项的边界处理

- 拖拽手柄隐藏（顺序调整无意义）
- ★ 星标隐藏（唯一菜单项自动为默认页）
- 左侧列表保留但唯一条目不可操作

##### 2.3.2 菜单项过多时的拖拽体验

- 拖拽到面板顶部边缘 → 自动向上滚动
- 拖拽到面板底部边缘 → 自动向下滚动
- 滚动速度随拖拽位置离边缘越近而加快

##### 2.3.3 多标签页/多设备并发编辑

- 最后保存者覆盖为准
- 检测到并发写入冲突时提示「检测到菜单设置已被其他窗口修改，请刷新后重试」

##### 2.3.4 弹窗与面板的关闭方式

支持：✕ 按钮 / 遮罩点击 / ESC 键 / 取消按钮

### 3. 接口需求

#### 3.1 接口总览

| 序号 | 接口 | 输入 | 输出 | 已有接口？ | 优先级 | 说明 |
|------|------|------|------|----------|-------|------|
| API1 | 菜单列表接口 | 用户Token | 权限内可见菜单项列表（含默认顺序、默认页） | ✅ 已有 | P0 | 门户引擎已有，需确认字段 |
| API2 | 用户菜单设置保存接口 | 用户ID + 菜单顺序 + 默认页 | 保存结果 | 🆕 新建 | P0 | V1.0 核心——保存用户自定义 |
| API3 | 用户菜单设置查询接口 | 用户ID | 用户自定义菜单顺序、默认页 | 🆕 新建 | P0 | 登录时读取个人设置 |
| API4 | 后台组默认页接口 | 用户ID / 用户组ID | 该组的默认页设置 | ✅ 已有 | P0 | 门户引擎后台已支持，需确认字段 |
| API5 | 个人信息接口 | 用户Token | 账号/手机/邮箱等个人信息 | ✅ 已有 | P0 | 个人设置面板展示用 |
| API6 | 个人信息保存接口 | 用户ID + 字段 + 值 | 保存结果 | ✅ 已有 | P0 | 个人手机/座机行内编辑保存 |
| API7 | 页面布局保存接口 | 门户ID + 布局配置 | 保存结果 | ✅ 已有 | P0 | 复用门户引擎已有能力 |

#### 3.2 排序规则

菜单项展示顺序按以下层级递进（权限过滤为前提）：

| 层级 | 维度 | 说明 |
|------|------|------|
| 第一层 | 用户自定义 | 用户在设置模式中保存的顺序 |
| 第二层 | 后台默认排序 | 无个人设置时，按门户引擎后台配置的顺序 |

#### 3.3 默认页规则

| 层级 | 来源 | 说明 |
|------|------|------|
| 第一层 | 用户个人设置 | 用户在菜单设置模式中自行选择的默认页 |
| 第二层 | 后台用户组设置 | 管理员在后台为某一用户组设置的默认页 |
| 第三层 | 系统兜底 | 无设置时默认展示集团门户（全员有权限） |
