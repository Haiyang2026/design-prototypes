# 流程需求文档技能包

## 技能包结构

| 层级 | 文件 | 用途 |
|------|------|------|
| 🧭 路由 | `SKILL.md` | 当前文件。元定义 + 路由表，定义触发场景和目录结构 |
| 📖 规则 | `references/流程规则分层框架.md` | 5层决策框架：角色取值→跳过规则→审批方式→校验异常→接口推送 |
| 📖 规则 | `references/字段与审批规范.md` | 字段分组规则、来源填写规范、审批步骤配置规范、命名规范 |
| 📖 规则 | `references/文档协作规范.md` | 文档编写流程、修改处理流程、封版机制、缺陷/导入规则 |
| 📋 模板 | `templates/流程需求文档模板.md` | 标准文档模板（0-概述→1-字段→2-流程→3-审批助手→附录） |
| 📋 模板 | `templates/变更记录模板.md` | 变更记录文档模板（变更概述→变更详情→审批流程变更→确认记录） |

## 触发场景

### 场景一：创建新流程需求文档
用户给出录音/业务描述 → 加载本 skill → 使用模板生成文档初稿 → 按协作流程逐项确认

### 场景二：逐项确认字段/审批流
按顺序：概述 → 字段分组 → 各分组字段 → 审批步骤 → 通知配置 → 外围接口
每步参考 `流程规则分层框架.md` 中的决策规则

### 场景三：修改/更新文档
遵循文档协作规范中的修改处理流程：接收→冲突判断→确认→执行（版本记录+变更文档+更新原文档）

### 场景四：生成审批流程图
全部定稿后，调用「审批流程图绘制」skill 生成 SVG 并嵌入文档

### 场景五：生成表单原型页面
文档全部确认定稿后，生成对应的表单原型 HTML 页面。规则如下：

**页面风格**：JMBOS 后台系统风格（顶导42px/侧边180px/品牌色#0C328C），参照 `jmbos-uiue-design-system` skill 表单页模板。

**内容分层**：
| 层次 | 内容 | 样式 |
|------|------|------|
| 表单主内容 | 用户可见的字段标签、输入框、占位符 | 统一黑色文字 `#141b24` |
| 用户提示 | 表单上给用户看的提示（如"非必填""尽量提供材料物性表"） | 灰色小字 `#7C8493`，紧跟在字段标签后 |
| 开发备注 | 给开发看的技术细节（组件类型、校验规则、权限说明） | 每个分组底部折叠面板，虚线边框 `#FF813B`，点击展开 |

**布局规范**：
- **严格列数**：每行只用 4 列或 2 列，**绝对禁止 3 列**（上下对不齐）。3 个字段时拆分 2+1 两行
- **内容多时 2 列**：字段标签或输入内容较长时，一行 2 个字段，不硬塞 4 列
- **全宽字段**：多行文本（textarea）、附件上传等宽字段独占一整行；`w:2` 标记的字段占 2 列
- **合并字段**：短字段合并为一列（如"工号 / 岗位"），用 `t:'双字段'` + `.dc` 横向排两个 input，中间 `/` 分隔
- **子表**：`dataTable()` 函数统一生成，含工具栏（新增行 + 导入 + 导出） + 操作列（逐行删除）；`addRow` 自动复制列数，`delRow` 保障至少保留 1 行
- **控件高度统一**：所有 `.in`（文本/日期）、`.sel`（下拉）统一 `height:32px;box-sizing:border-box`，视觉高度一致；`.ta`（多行文本）用 `min-height:80px`，仅多行文本场景使用
- **实现方式**：`gridRows()` 跟踪 `curW`（累计 w 值），满 4 列开新行，w:2 字段占位 2；`renderRow()` 通过 `.fi{flex:1}` + `.fi.fw2{flex:2}` + `.fi.fw{flex:0 0 100%}` 控制宽度

**环节字段权限展示**：
- 当前环节可编辑字段 → 正常样式，placeholder 显示填写提示
- 后续环节字段 → 整体 `opacity:0.55` + disabled，标签旁标注归属环节（蓝色小标签"分配工单""提交报告"等）
- 纯审批环节 → 全部字段 disabled

**必填标记**：`*` 号红色前置，非必填用 `（非必填）` 灰色文字

**文件命名**：`{流程名称}表单.html`，存放在需求文档同级目录
