# 工作收口助手 · Workday Wrapup

日常协助用户（海洋）在每天工作结束前做一次 5 分钟工作收口，生成「脱敏工作日报」并存入 Get 笔记。

## 触发方式

1. **主动触发**：每天下午 17:00-18:00 之间（或感知到用户当天工作已收尾时），主动说：「时间差不多了，来做今天的5分钟工作收口？」
2. **被动触发**：用户说「收口」「日报」「今日总结」「复盘今天」等

## 核心流程

### Step 1: 收集信息

**情况 A — 用户当天有较多对话记录**
直接从对话内容和当天的 `memory/YYYY-MM-DD.md` 日志中提取信息，生成日报。

**情况 B — 用户很忙，没有详细输入**
先问 5 个问题：
1. 今天最重要的一件工作是什么？
2. 今天哪里最卡？
3. 今天有没有重复劳动？
4. 今天有没有一个场景适合 AI 提效？
5. 明天最重要的一件事是什么？

根据回答生成完整日报。

### Step 2: 处理当天生成的 HTML 原型

如果在当天对话中有**新生成的 HTML 原型页面**（如交互 Demo、流程说明页等），按以下流程处理：

1. 将 HTML 文件 **push 到 GitHub 仓库** `Haiyang2026/design-prototypes`
   - 用 GitHub API PUT `/repos/Haiyang2026/design-prototypes/contents/{文件名}`
   - commit message 格式：`feat: 添加{说明}`
   - Token 存储在本地 mcp.json 中
2. 生成 GitHub Pages 链接：
   `https://haiyang2026.github.io/design-prototypes/{文件名}`
3. 将链接**写入日报的「今天最重要的产出」模块**中
4. 同步将链接**存到 Get 笔记**，作为该原型的访问入口

### Step 3: 生成日报

从 `references/daily-report-format.md` 加载完整格式模板，按 11 个模块生成。

**每次生成日报时，必须检查以下行为准则：**
1. **脱敏** — 不用真实客户名、项目名、人名、合同金额、内部系统名。用「项目A」「部门B」「领导L」「同事M」「流程系统X」代替
2. **不写流水账** — 只记录有价值的判断、产出、卡点、AI机会
3. **所有内容脱敏** — 不复制原文/邮件/聊天/工单内容

### Step 3: 写入 Get 笔记

日报正文生成完成后，调用 Get 笔记 skill 的保存功能：
- 使用 `note_type: plain_text`
- 标题格式：`工作日报｜YYYY-MM-DD｜关键词`
- 标签：`#工作日报 #AI落地 #流程运维 #邮件助手 #职业资产 #Hook闭环 #脱敏记录`

### Step 4: 告知完成

保存成功后告知用户：「今日日报已存入 Get 笔记 ✅」
