### 0. 概述

**目的**：CAE分析申请流程用于管理CAE仿真分析需求的提交与审批，申请人填写基础信息与分析目的，经三级组织负责人审批后，由工艺工程师分配工单（指定CAE分析工程师及预估完成日期），分析工程师完成仿真并填写分析结果后提交报告，最终经三级组织负责人审核及申请人确认。

| 项目 | 内容 |
|------|------|
| 流程ID | RD02004 |
| 业务对象 | WO_RD02004 |
| 流程图 | 见 2.1.1 审批流程图 |
| 流程描述 | CAE分析申请流程用于管理CAE仿真分析需求的提交与审批。申请人提交基础申请信息→三级组织负责人审批→分配工单（工艺工程师指定CAE分析工程师）→分析工程师提交报告（多人并审）→分析工程师三级组织负责人审核→申请人确认 |
| 流程分类 | 研发管理 > 研发设计 |
| 适用范围(组织) | 全员 |

### 2. 流程

#### 2.1 审批跳过规则（通用规则）

> **核心原则**：环节中若有字段需要填写，则该环节**不允许任何形式的跳过**。否则无人填写内容。

| 环节 | 有字段需填写？ | 为空跳过 | 相同审批人自动跳过 | 说明 |
|------|---------------|----------|--------------------|------|
| 1-三级组织负责人 | 否（纯审批） | 是 | 是 | 无填写内容，可正常跳过 |
| 2-分配工单 | **是**（CAE分析工程师、预估完成日期） | **否** | **否** | 必须有人填写，不可跳过 |
| 3-分析工程师提交报告 | **是**（实际分析问题类型、模拟依据、模拟条件及状态说明、分析结论、CAE分析报告） | **否** | **否** | 必须有人填写，不可跳过；多人并审 |
| 4-三级组织负责人审核 | 否（纯审批） | 是 | 是 | 无填写内容，可正常跳过 |
| 5-申请人确认 | **是**（用户评价、实施状态） | **否** | **否** | 必须有人填写，不可跳过 |

> **通用规则**：环节内需要填写的字段，默认均为**必填**，除非明确说明为非必填。
