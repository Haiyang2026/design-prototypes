基于旧流程「手机业务办理」（流程ID: IT_01110）的本次更新，涉及表单字段调整及联动规则变更。

| 项目 | 内容 |

| 流程ID | IT_01110 |
| 流程名称 | 手机业务办理流程 |
| 变更版本 | V1.0 → V1.1 |
| 变更日期 | 2026-06-22 |
| 封版日期 | 2026-07-13 |
| 变更类型 | 表单字段调整 |

---

## 变更详情

### 变更一：领用信息分组 → 字段调整

**涉及字段**：【领用人】→ 变更为【实际申请人】

| 变更项 | 变更前 | 变更后 |
|--------|--------|--------|
| 字段名称 | 领用人 | **实际申请人** |
| 默认值 | 用户从全集团人员中选择 | **自动获取当前提单人** |
| 必填 | - | **是** |
| 修改范围 | 全集团选人 | 可修改，修改时**优先展示本部门成员** |

### 变更二：申请信息分组 → 申请类型新增选项 + 联动

**涉及字段**：【申请类型】（已有字段）

| 变更项 | 变更前 | 变更后 |
|--------|--------|--------|
| 可选选项 | 原有选项 | 新增「**手机及手机号归还**」 |

**新增联动字段**：【工作手机号】

| 属性 | 内容 |
|------|------|
| 展示条件 | 仅当【申请类型】选择「手机及手机号归还」时展示 |
| 字段类型 | 文本 |
| 是否必填 | 是 |
| 编辑权限 | 只读 |
| 默认值 | 实际申请人的办公手机号（系统获取） |

### 变更三：申请信息分组 → 申请说明填写联动规则

**涉及字段**：【申请说明填写】（已有字段，多行文本）

| 变更项 | 变更前 | 变更后 |
|--------|--------|--------|
| 展示条件 | 始终展示 | 仅当【申请类型】选择「手机及手机号归还」时**展示且必填** |
| 必填 | 非必填 | 条件必填（选择「手机及手机号归还」时必填） |

> 其他申请类型下，此字段规则保持不变。

### 变更四：新增「手机及手机号归还」审批路径

**变更描述**：当【申请类型】选择「手机及手机号归还」时，提交后走以下审批路径：

| 步骤 | 环节名称 | 说明 |
|------|----------|------|
| step 1 | 汇报关系上级审批 | 审批方式：单审，汇报关系-Z1（直接上级） |
| step 2 | IT设备专员确认 | 沿用原有环节及规则，保持不变 |

> 其他申请类型审批路径不变。

### 变更五：新增「手机及手机号归还」流程结束邮件通知

**变更描述**：IT设备专员确认后，流程结束时发送邮件通知。

| 项目 | 内容 |
|------|------|
| 触发时机 | IT设备专员确认后，流程结束时 |
| 通知方式 | 邮件 |
| 邮件内容 | <span style="color:red">待定</span> |
| 发送目标 | 实际申请人 |

### 变更六：新增流程结束外围系统推送（滴滴工号手机号解除绑定）

**变更描述**：IT设备专员确认并发送邮件通知后，系统向外部系统推送"解除滴滴工号的手机绑定"指令。

| 项目 | 内容 |
|------|------|
| 触发时机 | IT设备专员确认 → 发送邮件通知后 |
| 推送方向 | 向外部系统推送 |
| 推送内容 | 解除实际申请人的滴滴工号手机绑定 |
| 状态 | <span style="color:red">待开发查看接口是否支持</span> |
