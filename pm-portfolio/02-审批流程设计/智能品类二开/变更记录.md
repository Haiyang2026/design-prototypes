## 三、变更详情

### 变更项1：环节名称变更

| 项目 | 内容 |
|------|------|
| **原来逻辑** | 主流程步骤ID 3 环节名称为【工厂评审回复】 |
| **新的逻辑** | 主流程步骤ID 3 环节名称改为【等待评审回复】 |
| **同步影响** | 无。审批助手标题保持"工厂评审回复"不变，无需随环节名称同步 |
| **变更原因** | 环节名称更准确反映业务状态：发起提交后等待工厂评审回复 |
| **涉及配置** | 主流程2.1审批步骤总览 → 步骤ID 3 |

### 变更项2：明细表字段顺序调整（渠道代码与渠道互换）

| 项目 | 内容 |
|------|------|
| **原来逻辑** | 明细列表字段顺序为：序号 → 【渠道】→ 【渠道代码】→ ... |
| **新的逻辑** | 明细列表字段顺序调整为：序号 → 【渠道代码】→ 【渠道】→ ... |
| **交互逻辑** | 先选择【渠道代码】（选择弹窗，带代码和名称），自动带出【渠道】名称 |
| **变更原因** | 业务操作习惯，先选代码再带出名称为更自然的操作顺序 |
| **涉及配置** | 主流程1.3明细列表 → 字段排列顺序、1.1字段分组描述 |

### 变更项3：子流程审批助手限制

| 项目 | 内容 |
|------|------|
| **原来逻辑** | 子流程所有节点均可使用审批助手（跳转主流程表单），包括：【发起人】、【企划资料收集】、【南安陶瓷工厂对接人】、【南安陶瓷工厂领导】、【永春良瓷工厂对接人】、【永春良瓷工厂领导】 |
| **新的逻辑** | 仅【企划资料收集环节】保留审批助手；工厂环节（南安对接人、南安领导、永春对接人、永春领导）**去掉审批助手**，不可查看主流程表单 |
| **变更原因** | 工厂相互独立进行评审，不应查看主流程结论，避免互相影响 |
| **影响范围** | 南安陶瓷工厂对接人、南安陶瓷工厂领导、永春良瓷工厂对接人、永春良瓷工厂领导 |
| **涉及配置** | 子流程3.审批助手设置 → 显示时机、权限控制 |

### 变更项4：工厂环节多人并审（开发检查确认）

| 项目 | 内容 |
|------|------|
| **原来逻辑** | 子流程工厂环节文档已标注"并审"，但需确认实际配置 |
| **新的逻辑** | 开发需检查确认以下4个环节是否已设置多人并审：南安陶瓷工厂对接人、南安陶瓷工厂领导、永春良瓷工厂对接人、永春良瓷工厂领导 |
| **变更原因** | 工厂相互独立进行评审，需要多人会签确认 |
| **说明** | 文档已正确标注为"并审"，此变更为开发检查确认项，非文档修改项 |

### 变更项5：全部日期字段增加日期约束

## 四、审批流程

### 主流程审批流程（不变）

```
【发起人】→【直接主管】→【等待评审回复】→【评审确认】→【销产供总经理审核】→【流程结束】
```

**变更说明**：仅步骤3环节名称从【工厂评审回复】改为【等待评审回复】，流程结构不变。

### 子流程审批流程（不变）

```
【发起人(机器自动)】→【企划资料收集】→【南安陶瓷工厂对接人】→【南安陶瓷工厂领导】→【永春良瓷工厂对接人】→【永春良瓷工厂领导】→【流程结束】
```

**变更说明**：流程结构不变，仅审批助手权限调整（工厂环节去掉审批助手，企划资料收集保留）。
